# Codex CLI 专用配置

## Codex 特有要求

- 直接进入 ultrathink 模式，如果分析代码问题和修复 bug，启用 sequential-thinking
- **先验证再判断** — 遇到可疑信息，必须先用工具查证事实，再给出结论
- 使用 `spawn_agent` / `send_to_agent` 时默认按串行处理；只有任务严格独立、且主子双方不会写入同一文件或模块时，才允许并行推进
- `spawn_agent` 返回即表示子 Agent 首轮已经完成；如果主 Agent 下一步依赖这个结果，不要继续主线实现，先消费子 Agent 输出

## 最终回复要求

- 默认用短段落收尾，优先 2-4 句说清“改了什么、结果如何、还有什么限制”
- 除非用户要求细节，否则不要长篇复盘，不要重复中间排查过程
- 除非内容天然是列表，否则不要把最终回复写成大段项目清单
- 只有在确实有必要时才提文件名、测试项、后续建议，避免把最终回复写成 changelog
- 如果任务已完成，先直接给结论，再补 1 句验证结果；不要堆叠背景说明

## 缩写解释

- `y` = 是 (Yes)
- `n` = 否 (No)
- `c` = 继续 (Continue)
- `r` = 确认 (Review)
- `u` = 撤销 (Undo)
